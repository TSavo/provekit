## Architectural lessons

**The gate is doing what the gate is for.** Cross-kit drift is exactly the failure class the substrate exists to catch. Sugar's own conformance gate catching Sugar's own bugs is the dogfood working at full strength.

**The 3-5 minute gate runtime is a feature, not a defect.** The fix-and-restart loop is what's actually expensive when a bug ships; the gate itself is cheap.

**Architectural option (c) was empirically refined to (b)+(c) hybrid.** The strict-(c) hypothesis ("same lifter walks self-contracts") was empirically false in practice. `sugar-lift` and `mint-self-contracts` are distinct binaries producing distinct CIDs. The correct cut is two surfaces per kit (general-purpose + self-contracts). Historically, the Rust-side alias table routed `--kit=<lang>` to the self-contracts surface as the canonical "what does this kit attest about itself" answer; current ProveKit represents that shortcut as project/user config plus manifests. See issue #176 comment thread.

**Test isolation is its own substrate concern.** Integration tests sharing repo-level state were correct integration coverage, but parallel execution turned shared state into races. The fix is `serial_test` named locks scoped per file, not architectural redesign. These tests SHOULD exercise real repo state.

### Bug 1: Worktree-base bleed
Spawned four agents from a shell whose HEAD was on `feat/lift-openapi-protobuf` (a separate open PR). Each `git worktree add ... -b <new>` cut from HEAD, not main. Resulting branches carried the lift-openapi PR's commits as parents. PR diffs against main showed 23 files instead of 13. Symptom: zig PR #229 "added" `sugar-cli/src/cmd_mint.rs` and `Cargo.lock`/`Cargo.toml` rust changes that weren't the agent's work.

**Fix applied:** `git rebase --onto origin/main HEAD~1`: drops the parent commits, keeps the kit's single commit.

**Lesson:** agent dispatch prompts must mandate `git worktree add <path> origin/main -b <branch>` (not `git worktree add <path> -b <branch>`). The default behavior of branching from HEAD is footgun-shaped when the parent shell is mid-PR-review. Future prompt template will pin `origin/main` explicitly.

### Bug 2: Parallel-session collision
Sir's other Claude session and mine were dispatched against the same set of issues unaware of each other. Both shipped zig Side B (#227 and #229), both shipped java Side B (#228 and #233). Java collided cleanly (mine closed as duplicate). Zig _both merged_: Sir's package was named `sugar-proof-envelope-zig/`, mine `sugar-self-contracts/`. Different paths, no git conflict, both landed. Now: dead code in main (Sir's package isn't wired into Makefile). Filed #235 for cleanup.

**Lesson:** before dispatching an agent against an issue, check: is the issue currently OPEN with no in-flight PR? `gh pr list --search "<issue-number> in:body"`: if any PR is pointing at it, hold. Different-named packages will pass conflict detection but represent the same architectural slot.

### Bug 3: Conformance gate caught a real bug, again
8+ cross-kit bugs surfaced today were the kind that only a parallel-language byte-equivalence test catches. Each Side B agent had to round-trip its envelope output against rust + go golden fixtures; one mismatch = reject. The gate is earning its rent.
