# ProvekIt Launch Showcase: Empirical Results

These numbers are reproduced verbatim from `cargo run --release -p provekit-showcase -- benchmark --lattice /tmp/showcase-lattice --queries 10000` against a fixture lattice generated by `cargo run --release -p provekit-showcase -- generate --size 100000 --output /tmp/showcase-lattice`.

Hardware: MacBook Pro 16 inch (2019), 2.6 GHz 6-core Intel Core i7, 32 GB DDR4. macOS 25.3.0. Z3 4.15.4. Rust 1.x release profile, `opt-level = 3`, `lto = false`.

Seed: `0xC0FFEEBEEF` (default). The fixture is byte-for-byte reproducible at this seed.

## Fixture lattice

```
contracts:        100,000
implications:     1,000,000
total mementos:   1,100,000
on-disk bytes:    2,562,704,893  (~2.39 GiB, ~2.56 GB)
.proof files:     1,100,000  (one per memento)
sharding:         <aa>/<bb>/<cid>.proof  (first 4 hex chars of CID after prefix)
generation time:  ~5 minutes wall-clock (includes 1.1M small-file fsyncs)
```

Each memento is signed with an Ed25519 key derived from the fixture seed and the memento's index, so producer signatures are deterministic.

## Compression ratio

```
on-disk bytes:           2,562,704,893
bytes per query:         64
ratio:                   ~ 4.00 x 10^7
```

The on-disk lattice is forty million times the size of the bytes any one query needs to verify it. Adding more mementos increases the lattice; the per-query cost stays fixed.

## ASCII latency profile

Median latency by tier, log-spaced (each `#` is one order of magnitude on a base-10 log axis from 1ns to 1s):

```
                              1ns      1us      1ms      1s
                              |--------|--------|--------|
tier 1  (memcmp 64)           ##                            58 ns
tier 2  (lookup + ed25519)             ###                  66 us
tier 3  (z3 from scratch)                       ###         24 ms
```

The point of the figure: each tier sits at a different decade. Tier 1 dominates Tier 2 by a factor of ~1,000x; Tier 2 dominates Tier 3 by another ~360x. The protocol's design is to push as many queries as possible into Tier 1, then Tier 2; Tier 3 is the slow path that only runs when no cached fact resolves the query.

## Extrapolation to a hypothetical petabyte lattice

A petabyte (`10^15` bytes) of lattice at the same per-memento footprint observed above would carry on the order of `4 x 10^11` mementos: roughly four hundred billion signed claims. At that scale:

```
on-disk bytes:           ~10^15
bytes per query:         64
ratio:                   ~1.5 x 10^13
```

If we imagine the substrate growing further, to a hypothetical exabyte lattice (`10^18` bytes), the ratio approaches `10^16`. At all sizes the per-query cost is unchanged: sixty-four bytes of digest, one memcmp, optionally one signature verify and one solver invocation. The lattice tractability theorem (`blake3-512:b6d7c2772c2929294d7f516f79559bd292e44f51805a6bd6ea0ca7fe365b82ec96b86c434f53dfb003f5acd306533831dc0257e46ead4c7d71081f9f56ec6d07`) proves the asymptotic claim formally. The showcase confirms it operationally at the `10^6` scale; the per-tier cost has no dependency on `N` in either direction.

## Validation, not competition

These numbers do not exist to claim ProvekIt is faster than tool `X`. ProvekIt is not in the same category as tool `X`. The numbers exist to confirm that the protocol's stated bound (constant-size verification at any node in the DAG) is operationally true on commodity hardware in 2026. The bound is a theorem; the showcase is its empirical witness.

## Reproduction

```sh
git checkout feat/protocol-v1-contract
cd implementations/rust
cargo build --release -p provekit-showcase
./target/release/provekit-showcase generate --size 100000 --output /tmp/showcase-lattice --seed 0xC0FFEEBEEF
./target/release/provekit-showcase benchmark --lattice /tmp/showcase-lattice --queries 10000 --seed 0xBEEFF00D
```

Same seed, same numbers (within nanosecond noise on Tier 1, microsecond noise on Tier 2, millisecond noise on Tier 3). The fixture lattice is byte-identical across runs at the same seed.
